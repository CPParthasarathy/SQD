# B1.3 Developer Setup Guide

## 1. Purpose

This guide defines the controlled Windows workstation setup for building,
flashing, monitoring and debugging the ESP32-S3 firmware.

## 2. Controlled project baseline

| Item | Controlled value |
|---|---|
| Host platform | Windows x64 |
| Project root | `D:\OneDrive\SQD` |
| ESP-IDF root | `D:\esp\v6.0.2\esp-idf` |
| ESP-IDF version | `v6.0.2` |
| Target | `esp32s3` |
| Validation board | Heltec WiFi LoRa 32 V3 |
| Validation project | `D:\OneDrive\SQD\verification\b1_2_minimal` |
| Shell | Windows PowerShell |
| VS Code extension | `espressif.esp-idf-extension@2.1.0` |

Versions not explicitly listed here must be recorded in the B1.1 tool-version
manifest.

## 3. Required software

The workstation requires:

1. Visual Studio Code.
2. ESP-IDF VS Code extension version 2.1.0.
3. Git for Windows.
4. Python 3.11.
5. ESP-IDF v6.0.2.
6. ESP-IDF-managed CMake and Ninja.
7. ESP-IDF-managed Xtensa compiler and debugger.
8. ESP-IDF-managed OpenOCD.
9. CP210x USB-to-UART driver where required by the board.
10. A data-capable USB cable.

Do not independently replace ESP-IDF-managed tools with unrelated system
versions.
