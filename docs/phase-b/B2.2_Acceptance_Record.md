# B2.2 Acceptance Record

## Scope

B2.2 establishes the controlled source-control and versioning process for the SQD ESP32-S3 firmware repository.

## Accepted controls

- Default integration branch is `main`.
- Changes to `main` require pull requests.
- Squash merging is the approved merge method.
- Merge commits and rebase merging are disabled.
- Linear history is required after governance activation.
- Force pushes and branch deletion are restricted.
- Pull-request conversations must be resolved.
- Conventional Commits govern commit subjects.
- Semantic Versioning governs firmware versions.
- Release tags use annotated `vMAJOR.MINOR.PATCH` format.
- Hotfix and emergency-exception processes are documented.
- Generated files and sensitive material remain excluded.
